# Impeccable — Design Guidelines

## Design Context

### Users
Angular developers (and the teams around them) who use `@ngx-translate` and are
tired of hunting through nested JSON files to find the key behind a piece of UI
text. They reach this landing page while evaluating a dev tool — they want to
understand *what it does*, *see it in motion*, and *copy the install steps* for
both the client library and the dev-server plugin. Context: a developer at their
desk, skim-reading, allergic to marketing fluff, wanting proof and code.

### Brand Personality
Confident, calm, premium. Three words: **crafted, exact, trustworthy.** The
voice is plain and technical — it shows rather than sells. It respects the
reader's intelligence and never over-explains. Emotional goal: the quiet
confidence of a well-made tool ("this was built by someone who cares").

### Aesthetic Direction
**Editorial / developer-craft** — like a beautifully typeset technical magazine.
- **Theme:** Light primary surface (warm paper / cream), with a dark "ink"
  section reserved for the live-demo and code blocks for dramatic contrast.
- **Color:** Tinted neutrals (warm paper, true ink — never pure #000/#fff). A
  single sharp accent: **amber/ochre** for highlights, the "live" marker, and
  primary actions. Optionally a secondary cool ink-teal for code syntax.
- **Type:** A distinctive **serif display face** for headlines paired with a
  clean **grotesque** for body (NOT Inter/Roboto/Arial). Monospace ONLY inside
  real code blocks — never as decorative "developer vibes."
- **Anti-references:** generic indigo-on-dark SaaS templates; purple→blue
  gradients; neon-on-dark; glassmorphism; identical icon-topped card grids;
  gradient text on headings/metrics. The current placeholder (indigo + Inter +
  dark) is explicitly what to move away from.

### Design Principles
1. **Show the product, not adjectives.** The hero and feature sections
   demonstrate hover → click → edit → save in context; code and diagrams carry
   the message more than prose.
2. **Typography is the design.** Strong modular scale, fluid `clamp()` sizing,
   real hierarchy via weight/size/serif-vs-grotesque contrast — minimal
   decoration.
3. **Tint every neutral; one accent only.** Warm paper and ink tones carry a
   subtle hue; amber is the single sharp accent, used sparingly for emphasis and
   actions.
4. **Asymmetric, editorial layout.** Break the grid intentionally, vary spacing
   rhythm, left-align text. Avoid centered-everything and uniform card walls.
5. **Make it fast and accessible.** WCAG AA contrast on the paper surface,
   honor `prefers-reduced-motion`, one orchestrated load reveal over scattered
   micro-animations.
