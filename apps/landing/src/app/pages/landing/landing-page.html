Why it feels effortless

Built to disappear into your dev loop.

The hard parts — finding the right key, writing the right file, never breaking your build — are handled. You just edit text.

Zero template annotations.

The client auto-tags translated elements at runtime and re-scans with a MutationObserver as the DOM and active language change. You add nothing to your markup.

Two packages, one seam

A browser half and a Node half that meet at the dev server.

@live-i18n/client BROWSER

A standalone Angular library wired in with provideLiveTranslations(...). It mounts the overlay, editor and toggle, auto-tags translated elements, resolves each key, and POSTs your edits back to the dev server.

  • → Runtime element tagging
  • → Invisible key markers + reverse-lookup
  • → Overlay, editor & toggle UI
@live-i18n/plugin NODE

A custom dev-server builder that wraps Angular's executeDevServerBuilder and mounts a middleware on POST /__live-i18n-update, safely rewriting your translation JSON on disk.

  • → Routes edits across searchRoots
  • → Path-traversal & prototype-pollution guards
  • → Preserves indentation & newline

Why a builder, not a Vite plugin? Angular's dev server runs Vite in-memory — there's no vite.config to register a plugin in, and AOT templates never pass through Vite's transform. The supported seam is the public executeDevServerBuilder API — a standard Angular CLI builder, so it runs in any Angular 17+ project, Nx or plain CLI.

Get started

Wired up in four steps.

One install, one builder swap, one provider. Then nx serve / ng serve and start editing — the steps work the same in an Nx workspace or a plain Angular CLI app.

1

Install both packages

Dev dependencies only — nothing reaches your bundle.

2 Plugin · Node

Point your serve target at the builder

It's a standard Angular CLI builder — register it in project.json for Nx, or angular.json for a plain CLI app — and point it at your *.json files. Add searchRoots for feature-split folders.

3 Client · Browser

Enable the inspector

Turn on key markers and register the live-translations provider in app.config.ts. Both are no-ops in production.

4

Serve and edit

Start the dev server, open the app, and hover any translated string. Edits write straight to disk.
